# Kivy APK 构建 - 快速开始

## 准备工作

### 1. 安装 Docker

1. 访问 [Docker 官网](https://www.docker.com/get-started)
2. 下载并安装 **Docker Desktop for Windows**
3. 安装完成后，启动 Docker Desktop
4. 等待 Docker 服务完全启动（右下角任务栏出现 Docker 图标）

### 2. 确认项目文件

确保项目目录中包含以下文件：
- `main.py` - 应用入口文件
- `kivy_calculator.py` - Kivy 应用主文件
- `buildozer.spec` - Buildozer 配置文件
- `Dockerfile` - Docker 配置文件
- `build_apk.bat` 或 `build_apk.ps1` - 构建脚本

## 开始构建

### 方法一：使用批处理脚本（推荐）

1. 打开文件资源管理器，进入项目目录
2. 双击运行 `build_apk.bat`
3. 等待脚本自动完成构建过程
4. 构建完成后，APK 文件会生成在 `bin` 目录中

### 方法二：使用 PowerShell 脚本

1. 打开 PowerShell，进入项目目录
2. 运行以下命令：
   ```powershell
   .\build_apk.ps1
   ```
3. 等待脚本自动完成构建过程
4. 构建完成后，APK 文件会生成在 `bin` 目录中

### 方法三：手动执行 Docker 命令

1. 打开 PowerShell 或命令提示符，进入项目目录
2. 构建 Docker 镜像：
   ```powershell
   docker build -t kivy-builder .
   ```
3. 运行容器构建 APK：
   ```powershell
   docker run --rm -v "${PWD}/bin:/app/bin" kivy-builder
   ```
4. 构建完成后，APK 文件会生成在 `bin` 目录中

## 构建时间

- **首次构建**：可能需要 30-60 分钟，因为需要下载和编译大量依赖项
- **后续构建**：通常只需要 5-15 分钟

请耐心等待，不要中断构建过程。

## 安装 APK

1. 在 `bin` 目录中找到生成的 APK 文件（文件名类似于 `框数板数计算器-0.1-arm64-v8a_armeabi-v7a-debug.apk`）
2. 将 APK 文件传输到 Android 设备（通过 USB、蓝牙、邮件等方式）
3. 在 Android 设备上打开 APK 文件进行安装
4. 如果提示"未知来源"，请在设置中允许安装来自未知来源的应用

## 常见问题

### 1. Docker 未安装或无法运行

**解决方法**：
- 确认 Docker Desktop 已正确安装
- 确认 Docker 服务正在运行
- 重启 Docker Desktop 或电脑

### 2. 构建过程中断或失败

**解决方法**：
- 检查网络连接是否稳定
- 确保有足够的磁盘空间（至少 10GB）
- 重新运行构建脚本

### 3. APK 文件未生成

**解决方法**：
- 查看构建日志，了解具体错误信息
- 确认所有项目文件完整
- 检查 `bin` 目录权限

### 4. 安装 APK 时提示"应用未安装"

**解决方法**：
- 确认 APK 文件完整下载
- 检查 Android 版本是否兼容（最低 Android 5.0）
- 卸载旧版本后重新安装

## 文件说明

| 文件 | 说明 |
|------|------|
| `main.py` | 应用入口文件，导入并运行 Kivy 应用 |
| `kivy_calculator.py` | Kivy 应用主文件，包含界面和逻辑 |
| `buildozer.spec` | Buildozer 配置文件，定义应用信息和构建设置 |
| `Dockerfile` | Docker 配置文件，定义构建环境 |
| `build_apk.bat` | Windows 批处理构建脚本 |
| `build_apk.ps1` | PowerShell 构建脚本 |
| `bin/` | 生成的 APK 文件存放目录 |

## 技术支持

如果在构建过程中遇到问题，可以：
1. 查看构建日志了解详细错误信息
2. 参考 `buildozer_guide.md` 和 `simple_build_guide.md` 获取更多信息
3. 访问 [Kivy 官方文档](https://kivy.org/doc/stable/)
4. 访问 [Buildozer 官方文档](https://buildozer.readthedocs.io/)
